# Implementation Plan: Phoneme Pronunciation Correction System (Design A)

This plan outlines the steps to implement a frame-level phoneme embedding architecture with CTC alignment, optimized for the NPTEL2020 dataset and H100 hardware constraints.

## Proposed Changes

---

### [Component] Core Model Implementation
We will move away from simple linear classification to a contrastive embedding space.

#### [NEW] `phoneme_embedder.py`
- Implement `Wav2Vec2PhonemeEmbedder` class.
- Add an `nn.Embedding` vocab layer for phonemes.
- Implement L2-normalization and cosine similarity matrix multiplication.
- Add a learnable `logit_scale` (temperature) parameter.
- Integrate CTC Loss calculation using these similarity logits.

---

### [Component] Training Pipeline
Optimizing for 50GB storage and 24-hour session limits.

#### [NEW] `train_streaming.py`
- Use `load_dataset(..., streaming=True)` for NPTEL2020.
- Implement a `DataCollatorCTCWithPadding` that handles variable length audio and labels.
- Configure `TrainingArguments` with:
    - `bf16=True` for H100 optimization.
    - `push_to_hub=True` for automatic cloud checkpointing.
    - `save_total_limit=2` to conserve local storage.
- Add logic to auto-resume from the last Hugging Face Hub checkpoint.

---

### [Component] Audio Preprocessing (VAD & FFT)
Ensuring clear audio for high-quality embeddings.

#### [NEW] `audio_utils.py`
- Implement `FourierFilter`: FFT-based noise reduction (spectral subtraction).
- Implement `SileroVADWrapper`: Neural Voice Activity Detection to crop silences.

#### [MODIFY] `train_streaming.py`
- Incorporate `audio_utils` into the `prepare_dataset` mapping.
- Ensure audio is cleaned before being passed to the model for embedding generation.

#### [MODIFY] `test_model.py`
- Add preprocessing step to microphone and file input loops.

---

### [Component] G2P Analysis & Optimization
Deep dive into phonemization performance.

#### [NEW] `g2p_analysis_report.md`
- Detailed report on current G2P dictionary performance.
- Evaluation of whether to train a joint acoustic-text G2P model.

---

### [Component] Inference and Analysis
Updating the scoring engine to leverage the new embedding-based outputs.

#### [MODIFY] `ScoreCalcs.py`
- Ensure the alignment logic correctly handles the output from the new embedder model.
- Optimize the `compute_scores` method for real-time feedback.

#### [MODIFY] `test_model.py`
- Update model loading to use the custom `Wav2Vec2PhonemeEmbedder` class.
- Update the inference loop to handle logits generated by cosine similarity.

## Verification Plan

### Automated Tests
- **Filter Validation:** Compare SNR (Signal-to-Noise Ratio) before and after FFT filtering on a noisy sample.
- **VAD Accuracy:** Run VAD on clips with known silence and verify correct cropping timestamps.
- **Shape Validation:** Verify that the similarity matrices have the correct `[Time, Vocab]` shapes.
- **Gradient Check:** Ensure the `logit_scale` parameter is actually learning during a small-batch run.

### Manual Verification
- **Record and Compare:** Use the `test_model.py` script to record a specific word (e.g., "because") and verify that the phoneme alignment correctly identifies intentionally forced mispronunciations.
- **Storage Monitoring:** Monitor disk usage during training to ensure it stays well below the 50GB limit.
